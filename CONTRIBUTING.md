# Contributing to the Health Equity WA Notebook

This is a living map. It is designed to get better with more eyes on it, especially from people with direct knowledge of First Nations health, impact investing, and community empowerment in WA and beyond.

---

## Ways to contribute

### Correct something
If a fact, figure, organisation description, or link is wrong or out of date --- open an issue with what is incorrect and what it should say, with a source if you have one.

### Add an organisation or node
If an organisation, program, finance mechanism, or pathway is missing, open an issue with:
- Organisation name and URL
- Which inclusion criteria it meets (health equity / education / community empowerment)
- A brief description (2–3 sentences)
- Any caveats that should be noted

### Add a new instance
If you want to adapt this framework for a different ecosystem, open an issue or fork the repo and we will work out how to structure it.

### Share your positionality
If you work in this space and want to contribute your own positionality statement or perspective, we welcome that. The hub is stronger when multiple voices are part of it.

---

## What gets included

Every addition must meet at least one of:
1. Health equity: active programs or investment towards First Nations or broader health equity outcomes
2. Education: investment in knowledge, capability, or workforce development at community level
3. Community empowerment: anything that increases community agency, structural capacity, or financial independence

Additions from First Nations researchers, community organisations, and people with direct on-the-ground knowledge are especially welcome and will be prioritised.

---

## What this is not

This hub is not a promotional directory and definitely not an endorsement. Organisations cannot pay or apply to be included. Inclusion reflects publicly available evidence of relevant work — nothing more.

---

## Code contributions

The hub is built in plain HTML/CSS/JavaScript with no framework dependencies — intentionally, so it stays accessible to non-developers. If you want to improve the UI, add a feature, or build a new instance, fork the repo and open a pull request.
